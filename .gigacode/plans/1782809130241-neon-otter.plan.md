- **Решение:** 
  - Добавлен параметр `onCheckpointComplete` в `PhotoCaptureScreen`
  - Вызов происходит при нажатии кнопки "Сохранить" в режиме предпросмотра

## 📊 Контроль последовательности (обновлено)

### Принцип работы:
```
1. Сканирование QR/NFC → запись SCAN (НЕ увеличивает индекс)
2. Диалог подтверждения (CHECKPOINT/QUESTION/INPUT/PHOTO) → увеличение индекса
3. Индекс обновляется только при подтверждении прохождения чекпоинта
```

### Типы чекпоинтов и увеличение индекса:

#### CHECKPOINT (обычная метка):
- QR/NFC сканирование → запись SCAN
- Диалог "Точка зафиксирована" → увеличение индекса при нажатии "ОК"

#### QUESTION (вопрос с вариантами ответов):
- QR/NFC сканирование → запись SCAN
- Выбор ответа → увеличение индекса

#### INPUT (ввод данных):
- QR/NFC сканирование → запись SCAN
- Нажатие "Сохранить" → увеличение индекса

#### PHOTO (съемка фото):
- QR/NFC сканирование → запись SCAN + переход на PhotoCaptureScreen
- Камера показывается (Preview + ImageCapture)
- Нажатие "Сделать фото" → сохранение в private папку + предпросмотр
- Нажатие "Сохранить" → копирование в галерею + вызов `onCheckpointComplete()` → **увеличение индекса**
- Нажатие "Сделать заново" → возврат к камере (индекс НЕ увеличивается)

## 🛠️ ИСПРАВЛЕНО: Запись времени завершения обхода в базу данных

**Проблема:** Метод `completeRound()` сохранял время завершения обхода в неправильном формате

- **Было:** Время сохранялось как отдельный ключ `"round_$roundId endTime"`, а не внутри JSON-строки обхода
- **Было:** Метод `loadRound()` искал `endTime` внутри JSON-строки, но не находил его, потому что он был сохранен как отдельный ключ
- **Проблема:** При отображении журнала время окончания обхода отображалось как "(не завершен)" вместо реального времени

**Решение:**
- Изменен метод `completeRound()` в `ShiftDatabaseManager.kt`
- Теперь метод загружает текущий обход, обновляет его и сохраняет в JSON-формате
- Время окончания теперь сохраняется внутри JSON-строки обхода и корректно загружается методом `loadRound()`

**Аналогичное исправление для `updateRoundProgress()`:**
- Изменен метод `updateRoundProgress()` для обновления `checkpointsPassed` и `sequenceViolations`
- Теперь метод корректно обновляет JSON-строку обхода
