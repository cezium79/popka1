## 📐 Технический подход

### REST API Yandex Cloud Storage

Yandex Cloud Storage использует S3-совместимый API. Для загрузки файла:

```
PUT /{bucket-name}/{file-path}
Host: storage.yandexcloud.net
Authorization: OAuth {oauth-token}
Content-Type: application/json
```

### Поток загрузки:

1. Пользователь нажимает "Загрузить в облако"
2. Проверяется наличие OAuth token в EncryptedSharedPreferences
3. Если token отсутствует → показывается CloudSettingsScreen
4. Пользователь вводит token и сохраняет
5. Генерируются отчеты (JSON + HTML)
6. Загружается JSON отчет в облако
7. Загружается HTML отчет в облако (или сохраняется локально для ссылки)
8. Показывается результат загрузки (Toast/Notification)
9. Логируется успех/ошибка

### Обработка ошибок:

- `401 Unauthorized` → token недействителен, очистить token и запросить заново
- `403 Forbidden` → недостаточно прав в бакете
- `404 Not Found` → бакет не найден
- `5xx` → временная ошибка сервера, показать сообщение об ошибке

### Реализованные методы:

1. **saveOAuthToken(token: String): Boolean** - Сохраняет OAuth token в SharedPreferences
2. **getOAuthToken(): String?** - Возвращает сохраненный OAuth token
3. **saveBucketName(bucket: String): Boolean** - Сохраняет имя бакета
4. **getBucketName(): String?** - Возвращает имя бакета
5. **saveBucketPath(path: String): Boolean** - Сохраняет путь в бакете
6. **getBucketPath(): String?** - Возвращает путь в бакете
7. **uploadFileToCloud(filePath: String, remoteFileName: String): Result<String>** - Загружает файл в Yandex Cloud Storage
8. **exportShiftReportWithCloud(shiftId: String, database: ShiftLogDatabase)** - Экспортирует отчет и загружает в облако
9. **Data class ExportResult** - Хранит результаты экспорта (jsonPath, htmlPath, cloudResult)

## 📝 Ключевые моменты

### Безопасность:
- OAuth token хранится в EncryptedSharedPreferences (шифрование AES)
- Token передается только по HTTPS
- Никакие credentials не логируются

### UX:
- Показывать прогресс загрузки (CircularProgressIndicator)
