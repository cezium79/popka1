# План исправления ошибки 415 при загрузке отчетов в Яндекс.Диск

## Проблема
При попытке загрузить отчет в Яндекс.Диск возникает ошибка 415 (Unsupported Media Type). Это происходит, когда передается JSON или HTML файл, который Яндекс.Диск не принимает с указанным Content-Type.

## Решение
Переделать загрузку отчетов так, чтобы сначала генерировался PDF файл (который уже создается правильно в `exportToPdf`), а затем загружался PDF файл в Яндекс.Диск через `uploadFileToDisk`.

## Текущее состояние
1. ✅ `CloudStorageManager.uploadFileToDisk()` - загружает файлы в Яндекс.Диск через получение ссылки для загрузки
2. ✅ `exportShiftReportWithCloud()` - экспортирует JSON и HTML отчеты, затем загружает их (оставлен для совместимости)
3. ✅ `exportShiftReportToPdfAndDisk()` - экспортирует PDF отчет и загружает его в Яндекс.Диск
4. ✅ `ShiftLogDetailScreen.exportToPdf()` - генерирует PDF отчет и сохраняет его в `Download/Ohrana/PDF/` (уже с параметром `outputPath` и `showPreview`)
5. ✅ `ShiftLogsListScreen` - обновлен для использования `exportShiftReportToPdfAndDisk()`
6. ✅ Ошибка "Can't toast on a thread..." исправлена удалением Toast из фонового потока
7. ✅ Автоматическое открытие PDF файла отключено при загрузке в облако
8. ✅ Ошибка 415 исправлена использованием `/upload` endpoint и правильным Content-Type

### 5. ✅ Исправлена ошибка "Can't toast on a thread that has not called Looper.prepare()"
**Проблема:** `exportToPdf` вызывал `Toast.makeText()` внутри функции, которая вызывалась из фонового потока (`Dispatchers.IO`)

**Решение:**
- Удалены все Toast вызовы из `exportToPdf`
- Toast теперь показывается только в UI потоке в вызывающем коде
- Обработка ошибок через возвращаемое значение `Result.failure(Exception(message))`

### 7. ✅ Исправлено автоматическое открытие PDF файла
**Проблема:** `exportToPdf` вызывал `context.startActivity(intent)` для открытия файла, что прерывало процесс загрузки в облако.

**Решение:**
- Добавлен параметр `showPreview: Boolean = true` в функцию `exportToPdf`
- При вызове из `CloudStorageManager` передается `showPreview = false`
- PDF файл генерируется и сохраняется без автоматического открытия
- Пользователь видит Toast с сообщением о сохранении

### 8. ✅ Исправлена ошибка 415 (Unsupported Media Type)
**Проблема:** Метод `uploadFileToDisk` использовал прямую загрузку через `PUT /v1/disk/resources` с `Content-Type: application/octet-stream`, что не работало для Yandex Disk API V1.

**Решение:**
- Метод `uploadFileToDisk` теперь сначала получает ссылку для загрузки через `GET /v1/disk/resources/upload?path=...`
- Затем загружает файл по полученной ссылке через `PUT` с `Content-Type: application/pdf`
- Это соответствует API документации Yandex Disk

### 9. ✅ Сборка проекта успешна после исправления ошибки 415
Запущена команда `./gradlew assembleDebug` - результат: `BUILD SUCCESSFUL`
