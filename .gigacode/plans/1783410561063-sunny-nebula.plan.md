# Plan: Fix Checkpoint Questions, Answers and PHOTO Display in Journal

## Problem Analysis

Checkpoint questions and answers were appearing empty in JournalScreen because:
1. `questionText` and `inputTitle` were not being fetched from checkpoint database
2. PHOTO checkpoints were not displaying correctly with photoPath
3. Three-level violation detection needed implementation (foreign checkpoint, outside route, out of sequence)

## Root Cause

1. **JournalScreen.kt**: Using placeholder logic for questionText/inputTitle without fetching from database
2. **ShiftDatabaseManager.kt**: Missing `getCheckpointById()` method usage in `updateLastScanEntry()`
3. **QrHandler.kt**: Missing `isCheckpointInRoute` check to distinguish violation types
4. **Photo handling**: Photos saved but not displayed correctly in journal

## Implementation Plan

### 1. Fixed JournalScreen.kt - Photo Viewing Functionality

**File:** `app/src/main/java/com/example/ohrana/JournalScreen.kt`

### 2. Implemented PhotoDialog Component

**File:** `app/src/main/java/com/example/ohrana/JournalScreen.kt`

#### Changes:

  - Provides "Закрыть" (Close) button

### 3. Fixed Three-Level Violation Detection

**File:** `app/src/main/java/com/example/ohrana/QrHandler.kt`

#### Changes:
- Added `isCheckpointInRoute` check using `PatrolRoute.containsCheckpoint()` method
- Distinguished three violation types:
  1. **Чужеродная метка** - checkpoint not in database at all
  2. **Чекпоинт вне маршрута** - checkpoint exists but not in current route
  3. **Точка вне очереди** - checkpoint in route but not next expected

#### PatrolRoute extension:
```kotlin
fun containsCheckpoint(checkpointId: String): Boolean {
    return checkpoints.contains(checkpointId)
}
```

### 4. Updated ShiftDatabaseManager

**File:** `app/src/main/java/com/example/ohrana/ShiftDatabaseManager.kt`

#### Changes:
- Implemented `getCheckpointById()` method to fetch checkpoint data from SharedPreferences
- Modified `updateLastScanEntry()` to:
  - Fetch questionText and inputTitle from checkpoint database
  - Skip updating SCAN records with `isSequenceCorrect == false` (violations)
  - Preserve checkpoint metadata when updating records

### 5. Updated OhrannikCabinetScreen

**File:** `app/src/main/java/com/example/ohrana/OhrannikCabinetScreen.kt`

  - "Точка вне очереди" - sequence violation

## User Flow

### Viewing Photo in Journal:
1. User navigates to JournalScreen
2. Finds PHOTO checkpoint entry in journal table
3. Photo icon (👁️) appears in column 6
4. User clicks icon
5. PhotoDialog opens with full-size photo
6. User can close dialog to return to journal

### Three Violation Types:
1. **Чужеродная метка**: QR/NFC code not found in database
2. **Чекпоинт вне маршрута**: Checkpoint exists but not in current route
3. **Точка вне очереди**: Checkpoint in route but not the expected next one
