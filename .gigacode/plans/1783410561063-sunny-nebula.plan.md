# Plan: Three-Level Violation Detection for Checkpoint Identification

## Problem Analysis

The system needed proper three-level violation detection for checkpoint identification:
1. **Foreign checkpoint** - QR/NFC code not found in database
2. **Outside route** - checkpoint exists but not in current route
3. **Out of sequence** - checkpoint in route but not next expected

Additionally, several display issues in JournalScreen needed fixing.

## Solution Summary

### Core Implementation:

1. **Added `SequenceErrorType` enum** - Three violation types: FOREIGN_CHECKPOINT, OUTSIDE_ROUTE, OUT_OF_SEQUENCE
2. **Updated `ShiftLogEntry` model** - Added `sequenceErrorType` field
3. **Updated `ShiftDatabaseManager`** - Save/load `sequenceErrorType` in database
4. **Updated `QrHandler`** - Pass correct `sequenceErrorType` for all violation types
5. **Updated `JournalScreen`** - Display violation types in "Примечание" column
6. **Updated other screens** - CloudStorageManager, OhrannikCabinetScreen, ShiftLogDetailScreen

### UI Changes:

- **JournalScreen**: 
  - Column "Примечание" now shows: "Чужая метка", "Вне маршрута", "Вне очереди"
  - CHECKPOINT violations show "-" (not "Точка пройдена")
  - PHOTO violations show "Фото снято"
  - Removed "Дата" column
  - Added "Сотрудник" column after "Время"

## Technical Details

### Database Model Changes:

```kotlin
enum class SequenceErrorType {
    NONE,
    FOREIGN_CHECKPOINT,    // Чужеродная метка
    OUTSIDE_ROUTE,         // Вне маршрута
    OUT_OF_SEQUENCE        // Вне очереди
}

data class ShiftLogEntry(
    // ... existing fields ...
    val sequenceErrorType: SequenceErrorType = SequenceErrorType.NONE
)

data class SequenceViolation(
    // ... existing fields ...
    val sequenceErrorType: SequenceErrorType = SequenceErrorType.OUT_OF_SEQUENCE
)
```

### Data Flow:

1. **Scan QR/NFC** → QrHandler parses code
2. **Check existence** → Not in database → FOREIGN_CHECKPOINT
3. **Check route** → In database but not in route → OUTSIDE_ROUTE
4. **Check sequence** → In route but wrong order → OUT_OF_SEQUENCE
5. **Save log** → With correct `sequenceErrorType`
6. **Display** → JournalScreen shows appropriate message

### Violation Display:

| Type | Message | Photo Display |
|------|---------|---------------|

| OUT_OF_SEQUENCE | "Вне очереди" | "-" |
| PHOTO + violation | N/A | "Фото снято" |
