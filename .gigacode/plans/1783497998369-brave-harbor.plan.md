# План реализации: Сохранение данных смены в HTML файл отчета

## 📋 Обзор задачи
При закрытии смены необходимо автоматически сохранять все данные (обходы, сотрудники, временные метки, нарушения) из временной памяти в постоянное хранилище в виде JSON-файла отчета для хранения в памяти телефона и последующей загрузки в Яндекс Диск в HTML формате.

---

## 🧠 Контекст (из анализа кода)

### Текущая структура данных:
1. **ShiftDatabaseManager.kt** - внутренняя база данных смен:
   - `ShiftRecord` - информация о смене (ID, время начала/окончания, список охранников)
   - `RoundRecord` - информация об обходах (ID, статус, время)
   - `ShiftLogEntry` - записи сканирования (checkpoint, employee, answer, photo)
   - `SequenceViolation` - нарушения последовательности

2. **SharedPrefsManager.kt** - управление SharedPreferences:
   - `closeCurrentShift()` - закрытие смены
   - `loadGuards()` - список охранников
   - `loadRouteAlarms()` - список будильников/обходов

3. **Очередь закрытия смены:**
   - Пользователь нажимает кнопку закрытия → `closeCurrentShift()` → генерация JSON → сохранение

4. **В Журнале завершенных смен:**
   - Две кнопки для загрузки в Яндекс Диск:
     - "Загрузить PDF в Яндекс.Диск" - загружает PDF файл
     - "Загрузить HTML в Яндекс.Диск" - загружает HTML файл (новая кнопка)

### Куда сохранять:
- JSON файлы: `/Download/Ohrana/Reports/` (стандартная папка загрузок)
- HTML файлы: загружаются в Яндекс Диск по пути `Ohrana/shift_[shiftId]_[timestamp].html`
- Формат: JSON (структурированный) для хранения, HTML (визуальный) для отображения

```kotlin
fun closeCurrentShift() {
    val endTime = dateFormat.format(Date())
    val activeShiftId = prefs.getString("active_shift_id", null)
    
    // ГЕНЕРАЦИЯ ОТЧЕТА - JSON файл для хранения
    activeShiftId?.let { shiftId ->
        val cloudManager = CloudStorageManager(context)
        val result = cloudManager.exportShiftReport(shiftId, shiftDatabase)
        if (result != null) {
            Log.d("ReportManager", "Отчет сохранен в: $result")
        } else {
            Log.e("ReportManager", "Ошибка генерации отчета")
        }
    }
    
    // ОСТАЛЬНЫЕ ДЕЙСТВИЯ ЗАКРЫТИЯ
    // ... существующий код
}
```

**Механизм:**
- При закрытии смены вызывается `CloudStorageManager.exportShiftReport()`
- Генерируются оба файла: JSON и HTML
- JSON сохраняется в `/Download/Ohrana/Reports/`
- HTML файл создается для последующей загрузки в Яндекс Диск

4. **app/src/main/res/xml/file_paths.xml** - добавлен путь для отчетов

### Как работает:
1. При закрытии смены вызывается `closeCurrentShift()`
2. Метод извлекает ID активной смены
3. Генерирует полный отчет через `ReportManager`
4. Конвертирует в HTML с CSS стилями
5. Сохраняет HTML файл в папку `/Android/media/com.example.ohrana/Ohrana/Reports/` или fallback в папку приложения
6. Логирует результат (успех/ошибка)

### Формат отчета:
HTML файл с встроенными CSS стилями, содержащий:
- Заголовок отчета
- Метаинформацию (ID смены, время генерации)
- Таблицу охранников
- Секцию обходов с прогрессом
- Таблицу логов сканирования
- Таблицу нарушений последовательности
- Подвал с информацией об устройстве

### Обработка ошибок:
- Ошибки сохранения логируются через Log.e()
- Закрытие смены не блокируется при ошибке
- Файлы сохраняются в папку приложения как fallback

### Имя файла:
`shift_report_[shiftId]_[timestamp].html`

Пример: `shift_report_NS080726_001_1783497998369.html`
