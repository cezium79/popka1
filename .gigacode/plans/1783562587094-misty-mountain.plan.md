# 📋 План реализации: Интеграция File.io как альтернативного облачного хранилища

## 🎯 Обзор задачи

Интегрировать File.io как альтернативное облачное хранилище для отчетов с настройками в интерфейсе администратора. File.io предоставляет анонимное временное хранение файлов без регистрации, файлы хранятся 14 дней.

---

## 🌐 File.io API

### Метод загрузки:
- **URL:** `POST https://file.io`
- **Content-Type:** `multipart/form-data` или `text/html`
- **Тело запроса:** HTML отчет в теле запроса
- **Ответ:** JSON с полями:
  - `success` (boolean) - статус загрузки
  - `link` (string) - URL для скачивания файла
  - `id` (string) - ID файла
  - `expires` (integer) - время жизни в секундах (14 дней = 1209600)

### Особенности:
- Анонимный сервис - не требует API ключа
- Срок хранения: 14 дней (1209600 секунд)
- Ограничение на размер файла: 5 GB
- Автоматическое удаление после срока действия

## 📂 Ключевые файлы

### 1. CloudStorageManager.kt
**Расположение:** `app/src/main/java/com/example/ohrana/CloudStorageManager.kt`

#### Новые методы:
- `uploadHtmlToFileIoDirect(shiftId: String, shiftDatabase: ShiftDatabaseManager): Result<String>` - загружает HTML отчет напрямую в File.io без сохранения на телефоне
- `exportHtmlToFileIo(shiftId: String, shiftDatabase: ShiftDatabaseManager): FileIoExportResult` - экспортирует и загружает HTML отчет
- `setFileIoEnabled(enabled: Boolean)` - включает/отключает использование File.io
- `isFileIoEnabled(): Boolean` - проверяет, включен ли File.io

#### Новые классы результатов:
- `FileIoExportResult` - результат экспорта HTML отчета и загрузки в File.io
  - `htmlPath: String?` - путь к локальному файлу (null если не сохранялся)
  - `fileIoResult: Result<String?>` - результат загрузки в File.io (URL или ошибка)
  - `isSuccess(): Boolean` - успешна ли загрузка
  - `getFileIoErrorMessage(): String` - сообщение об ошибке
  - `getFileIoUrl(): String?` - URL файла в File.io

### 2. SharedPrefsManager.kt
**Расположение:** `app/src/main/java/com/example/ohrana/SharedPrefsManager.kt`

## 🔗 Интеграция с существующими компонентами

### При закрытии смены (SharedPrefsManager.kt):
```kotlin
fun closeCurrentShift() {
    // ... существующий код ...
    
    // ГЕНЕРАЦИЯ ОТЧЕТА - JSON файл для хранения
    activeShiftId?.let { shiftId ->
        val cloudManager = CloudStorageManager(context)
        val result = cloudManager.exportShiftReport(shiftId, shiftDatabase)
        if (result != null) {
            Log.d("SharedPrefsManager", "Отчеты сохранены локально в: $result")
        }
        
        // Если File.io включен, загружаем отчет туда тоже
        if (isFileIoEnabled()) {
            val fileIoResult = cloudManager.exportHtmlToFileIo(shiftId, shiftDatabase)
            if (fileIoResult.isSuccess()) {
                val url = fileIoResult.getFileIoUrl()
                saveFileIoUrl(shiftId, url ?: "")
                Log.d("SharedPrefsManager", "File.io URL saved: $url")
            } else {
                Log.e("SharedPrefsManager", "File.io upload failed: ${fileIoResult.getFileIoErrorMessage()}")
            }
        }
    }
    
    // ... остальной код закрытия ...
}
```

## 📊 Преимущества File.io

1. **Анонимность** - не требует регистрации или API ключа
2. **Простота** - всего один POST запрос без сложной авторизации
3. **Быстрота** - мгновенная загрузка без настройки
4. **Бесплатно** - базовый функционал бесплатен
5. **Временное хранение** - файлы удаляются автоматически через 14 дней

---

## 🔒 Безопасность

- Файлы хранятся 14 дней, затем автоматически удаляются
- Нет хранения персональных данных на сторонних серверах
- URL файла не содержит чувствительной информации
- Возможность очистить URL из SharedPreferences при необходимости
