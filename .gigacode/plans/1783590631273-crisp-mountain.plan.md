# План полной очистки от File.io и переход на Яндекс.Диск API

## Цель
Полностью удалить File.io из проекта и перейти на Яндекс.Диск API для отправки HTML отчетов из Android приложения.

## Требования
- Удалить все файлы и методы File.io из проекта
- Оставить методы Яндекс.Диска и DropMeFiles без изменений
- Убедиться, что проект успешно компилируется
- Проверить отсутствие ссылок на File.io во всем коде

## Ключевые знания
- File.io API endpoint: https://file.io (удален из проекта)
- File.io не поддерживает мобильные приложения (возвращает HTML вместо JSON)
- OkHttp API: `RequestBody.create(contentType, content)` - правильный порядок аргументов
- MIME-типы: "application/json", "text/html", "application/octet-stream"
- MultipartBody.FORM автоматически устанавливает Content-Type с boundaries
- Поле в multipart должно называться строго "file"
- Yandex Disk API endpoints: https://cloud-api.yandex.ru
- Yandex Cloud Storage: storage.yandexcloud.net
- Build command: .\gradlew.bat assembleDebug
