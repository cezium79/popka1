# План реализации: Отображение количества чекпоинтов в расписании обходов

## 📋 Описание задачи

В окне **Расписание обходов** (ScheduleScreen) в разделе 2 "Расписание и временные рамки" необходимо:

1. **Для каждого обхода** - отобразить количество чекпоинтов в используемом маршруте рядом с переключателем "Игнор"
2. **В конце раздела 2** - отобразить общее количество чекпоинтов за все обходы
3. **При изменении** расписания или состава маршрутов - значение должно автоматически пересчитываться
4. **Сохранять в память** общее количество чекпоинтов

---

## 🔍 Анализ текущей структуры

### Файл: `ScheduleScreen.kt`

**Текущая структура:**
- Загружает `routeAlarms` (список будильников/обходов)
- Загружает `allRoutes` (список всех маршрутов)
- Отображает каждое поле времени и выпадающий список выбора маршрута
- Имеет переключатель `Switch` для включения/отключения обхода
- **Есть вычисляемое значение:** `roundCheckpointCounts` - количество чекпоинтов для каждого обхода
- **Есть вычисляемое значение:** `totalScheduledCheckpoints` - общее количество чекпоинтов

**Проблема:** Эти значения вычисляются, но **не отображаются** в UI.

### Файл: `SharedPrefsManager.kt`

**Ключевые методы:**
- `saveTotalScheduledCheckpoints(total: Int)` - сохраняет общее количество
- `loadTotalScheduledCheckpoints(): Int` - загружает сохраненное значение
- `loadRoutes(): List<Route>` - загружает все маршруты
- `getRouteById(id: String): Route?` - получает маршрут по ID

### Файл: `Marshruti.kt`

**Ключевая функциональность:**
- Редактирование чекпоинтов (добавление/удаление)
- Модификация маршрутов
- Сохранение в `SharedPrefsManager`

### Файл: `RouteEditorScreen.kt`

**Ключевая функциональность:**
- Создание/редактирование маршрутов
- Выбор чекпоинтов для маршрута
- Изменение порядка чекпоинтов
- **Уже отображает:** `route.checkpointIds.size` - количество точек в маршруте

### Шаг 3: Проверка логики пересчета

**Убедиться, что:**
- `roundCheckpointCounts` зависит от `routeAlarms` и `allRoutes`
- `totalScheduledCheckpoints` зависит от `routeAlarms` и `allRoutes`
- Сохранение происходит в `LaunchedEffect`

**Текущая реализация:**
```kotlin
// Количество чекпоинтов для каждого обхода
val roundCheckpointCounts = remember(routeAlarms, allRoutes) {
    routeAlarms.map { alarm ->
        alarm.routeId?.let { routeId ->
            allRoutes.find { it.id == routeId }?.checkpointIds?.size ?: 0
        } ?: 0
    }
}

// Общее количество чекпоинтов
val totalScheduledCheckpoints = remember(routeAlarms, allRoutes) {
    routeAlarms.map { alarm ->
        alarm.routeId?.let { routeId ->
            allRoutes.find { it.id == routeId }?.checkpointIds?.size ?: 0
        } ?: 0
    }.sum()
}
```

### Шаг 4: Проверка сохранения

**Убедиться, что в `LaunchedEffect` есть:**
```kotlin
sharedPrefsManager.saveTotalScheduledCheckpoints(totalScheduledCheckpoints)
```

## 🛠️ Исправление проблемы с отображением общего количества

**Проблема:** Общее количество чекпоинтов не отображалось в строке "Общее количество чекпоинтов за смену:"

**Причина:** Использование `by remember { derivedStateOf { ... } }` для `allRoutes` и `remember(routeAlarms, allRoutes)` для пересчета `totalScheduledCheckpoints` не обеспечивало своевременного обновления значений.

**Решение:**

1. **Заменен способ загрузки маршрутов:**
   ```kotlin
   // Было:
   val allRoutes by remember { derivedStateOf { sharedPrefsManager.loadRoutes() } }
   
   // Стало:
   val allRoutesList = sharedPrefsManager.loadRoutes()
   ```

2. **Использование `derivedStateOf` для `totalScheduledCheckpoints` и `roundCheckpointCounts`:**
   ```kotlin
   val totalScheduledCheckpoints by remember {
       derivedStateOf {
           routeAlarms.map { alarm ->
               alarm.routeId?.let { routeId ->
                   allRoutesList.find { it.id == routeId }?.checkpointIds?.size ?: 0
               } ?: 0
           }.sum()
       }
   }
   
   val roundCheckpointCounts by remember {
       derivedStateOf {
           routeAlarms.map { alarm ->
               alarm.routeId?.let { routeId ->
                   allRoutesList.find { it.id == routeId }?.checkpointIds?.size ?: 0
               } ?: 0
           }
       }
   }
   ```

**Преимущества:**
- `derivedStateOf` автоматически пересчитывает значения при изменении зависимостей
- `remember` кэширует вычисление, но обновляет при изменении зависимостей
- Используется `allRoutesList` напрямую, без промежуточного `by`

**Изменения:**
- Заменен `allRoutes by remember` на `allRoutesList = sharedPrefsManager.loadRoutes()`
- Заменен `remember(routeAlarms, allRoutes)` на `by remember { derivedStateOf { ... } }`
- Обновлены все ссылки на `allRoutes` на `allRoutesList`

**Результат:** Общее количество чекпоинтов теперь отображается корректно и обновляется при изменении маршрутов или расписания!
