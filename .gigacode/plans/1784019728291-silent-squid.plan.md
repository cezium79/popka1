## Решение проблемы с обновлением имени звука

### Корневая причина
Использование `remember(eventId)` кэширует значение URI звука, но когда `eventId` постоянен (1, 2 или 3), Compose не обнаруживает изменения URI, даже если звук был изменен через системное окно.

### Решение
Использование `mutableStateOf` для `currentSoundUri` и `soundName` с `LaunchedEffect` для отслеживания изменений:

```kotlin
var currentSoundUri by remember { mutableStateOf(prefsManager.getEventSoundUri(eventId)) }
var soundName by remember { mutableStateOf("...") }

LaunchedEffect(eventId) {
    currentSoundUri = prefsManager.getEventSoundUri(eventId)
}

LaunchedEffect(currentSoundUri) {
    soundName = try {
        val ringtone = RingtoneManager.getRingtone(context, currentSoundUri)
        ringtone?.getTitle(context)?.toString() ?: "Неизвестный звук"
    } catch (e: Exception) {
        // дефолтные имена для стандартных звуков
    }
}
```

Теперь:
1. `LaunchedEffect(eventId)` обновляет URI при изменении `eventId`
2. `LaunchedEffect(currentSoundUri)` обновляет имя звука при изменении URI
3. Имя звука всегда отображает текущий URI из `prefsManager`

### Передача SharedPrefsManager
Исправлена рассинхронизация между экземплярами `SharedPrefsManager`, создаваемыми в разных компонентах. Теперь `prefsManager` передаётся из родительской композиции в карточки и диалог:

```kotlin
fun SoundCardWithDropdown(..., prefsManager: SharedPrefsManager)
fun SoundSelectionDialog(..., prefsManager: SharedPrefsManager)
```

## Вариант реализации: Использование встроенных Android звуков

### 1. Создать SoundPlayer.kt (Утилита для воспроизведения звуков)
**Файл:** `app/src/main/java/com/example/ohrana/SoundPlayer.kt`

**Функции:**
- `init(context)` - инициализация системных звуков
- `release()` - освобождение ресурсов
- `shouldPlaySound(context, eventId)` - проверка настроек звука
- `playSuccess(context)` - звук успеха (событие 1)
- `playError(context)` - звук ошибки (событие 2)
- `playWarning(context)` - звук предупреждения (событие 3)
- `playCustomSound(context, eventId, uri)` - кастомный звук

**Звуки:**
- **Успех:** `RingtoneManager.TYPE_NOTIFICATION` (обычный звук уведомления)
- **Ошибка:** `RingtoneManager.TYPE_ALARM` (звук будильника)
- **Предупреждение:** `RingtoneManager.TYPE_RINGTONE` (звук звонка)

### 2. Интеграция в MainActivity
**Где:**
- `onCreate()` - инициализация SoundPlayer
- `onDestroy()` - освобождение ресурсов

### 3. Интеграция в PhotoCaptureScreen
**Где:**
- При ошибке съемки: `SoundPlayer.playError(context)`
- **ИЗМЕНЕНО:** Убран вызов звука успеха после сохранения фото (звук проигрывается только в диалоге с зеленой галочкой)
- **ИЗМЕНЕНО:** Добавлен диалог с зеленой галочкой после нажатия "Сохранить"

### 4. Интеграция в OhrannikCabinetScreen
**Где:**
- При успешном NFC-сканировании (показ диалога с зеленой галочкой): `SoundPlayer.playSuccess(context)`
- При успешном QR-сканировании (показ диалога с зеленой галочкой): `SoundPlayer.playSuccess(context)`
- При ошибке NFC-сканирования (тег не найден): `SoundPlayer.playError(context)`
- При нарушении последовательности: `SoundPlayer.playError(context)`

## Новые возможности

### Настройки звука для каждого типа событий
Теперь в `SoundSettingsScreen` есть три отдельных карточки для каждого типа звука:

1. **Звук успеха** (событие 1)
   - Включает/выключает звук при успешном сканировании
   - Использует системный звук уведомления (TYPE_NOTIFICATION)
   - Кнопка "play" для прослушивания
   - Кнопка "stop" для остановки воспроизведения
   - Кнопка "arrow" для выбора звука из выпадающего меню

2. **Звук ошибки** (событие 2)
   - Включает/выключает звук при ошибках
   - Использует системный звук будильника (TYPE_ALARM)
   - Кнопка "play" для прослушивания
   - Кнопка "stop" для остановки воспроизведения
   - Кнопка "arrow" для выбора звука из выпадающего меню

3. **Звук предупреждения** (событие 3)
   - Включает/выключает звук при нарушениях
   - Использует системный звук звонка (TYPE_RINGTONE)
   - Кнопка "play" для прослушивания
   - Кнопка "stop" для остановки воспроизведения
   - Кнопка "arrow" для выбора звука из выпадающего меню

Каждая карточка содержит:
- Переключатель для включения/выключения звука
- Кнопку "play" для прослушивания выбранного звука
- Кнопку "stop" для остановки воспроизведения
- Кнопку "arrow" для вызова выпадающего меню выбора звука

### Выбор звука
При нажатии на кнопку "arrow" автоматически открывается системное окно `RingtoneManager.ACTION_RINGTONE_PICKER`, которое предоставляет:
- Список всех доступных системных звуков (уведомления, будильники, звонки, почта, и т.д.)
- Возможность выбрать стандартный звук или любой другой доступный
- Автоматическое обновление имени звука при возврате

### Методы в SharedPrefsManager
Добавлены новые методы для работы с звуками:
```kotlin
fun getEventSoundUri(eventId: Int): Uri
fun setEventSoundUri(eventId: Int, uri: Uri)
```
