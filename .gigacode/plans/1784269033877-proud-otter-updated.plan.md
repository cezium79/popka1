## Изменения в плане

### 1. Исправление проблемы закрытия приложения после отправки email

#### Проблема
Приложение закрывалось после отправки письма с ссылкой на отчет через функцию `EMAIL_SMTP`.

#### Корневая причина
Неправильное управление корутинами и контекстами выполнения:
- `sendLinkViaSmtp` создавал новую корутину в `Dispatchers.Main` внутри уже существующей корутины
- Методы `sendSimpleEmail` и `sendEmailWithAttachment` вызывали `Toast.makeText()` из фонового потока

#### Решение
**Файл: `app/src/main/java/com/example/ohrana/SharedPrefsManager.kt`**
- Изменил `sendLinkViaSmtp` для создания корутины в `Dispatchers.IO`
- Добавил правильное переключение в `Dispatchers.Main` для UI-операций через `withContext`
- Добавил обработку исключений `try-catch`

**Файл: `app/src/main/java/com/example/ohrana/EmailManager.kt`**
- Убрал все `Toast.makeText()` из фоновых потоков в `sendSimpleEmail` и `sendEmailWithAttachment`
- Методы теперь возвращают `true/false`, а UI-сообщения отображаются в вызывающем коде

#### Результат
- Приложение больше не закрывается после отправки письма через SMTP
- Правильное управление корутинами без вложенных контекстов

### 2. Выравнивание календаря и красный цвет для выходных дней

#### Проблема
- Названия дней недели и числа не были четко выровнены по вертикали
- Дни Сб и Вс не были выделены красным цветом

#### Решение
**Файл: `app/src/main/java/com/example/ohrana/JournalScreen.kt`**

**Изменения заголовков дней недели:**
- Добавлен `textAlign = TextAlign.Center` для всех элементов
- Названия дней Сб и Вс теперь красного цвета `Color(0xFFFF6B6B)`

**Изменения чисел в календаре:**
- Исправлено определение дня недели для каждого числа
- Добавлен правильный расчет `adjustedIndex`:
  - `Calendar.DAY_OF_WEEK`: Воскресенье=1, Понедельник=2, ..., Суббота=7
  - Преобразование: `adjustedIndex = if (weekDayIndex == Calendar.SUNDAY) 6 else weekDayIndex - 2`
- Добавлено выделение выходных дней красным текстом

#### Результат
- Все элементы календаря четко выровнены
- Выходные дни (Сб, Вс) выделены красным цветом текста
- Правильное определение дня недели для каждого числа

### 3. Улучшение календаря: серые контуры и зеленый цвет выбранной даты

- Выбранная дата для фильтрации имеет зеленый контур (`0xFF4CAF50`)
- Будние дни — серый текст (`0xFF9E9E9E`)
- Выходные дни — красный текст (`0xFFFF6B6B`)
- Исправлен формат даты для корректного сравнения `selectedDateStr`

#### Результат
- Визуально понятно, какая дата выбрана для фильтрации
- Правильное отображение будних и выходных дней
- Серый контур для всех чисел, зеленый — для выбранной

### 4. Обработка системной кнопки "Назад"

#### Проблема
- При нажатии системной кнопки "Назад" приложение вылетало
- Кнопка "Назад" в интерфейсе работала, а системная — нет

#### Решение
**Файл: `app/src/main/java/com/example/ohrana/JournalScreen.kt`**

**Изменения:**
- Добавлен импорт `androidx.activity.compose.BackHandler`
- Добавлен обработчик `BackHandler` который вызывает `onBack()`

```kotlin
BackHandler {
    showMenu = false
    onBack()
}
```

#### Результат
- Системная кнопка "Назад" теперь работает как кнопка "Назад" в интерфейсе
- При нажатии сначала скрывается всплывающее меню, затем вызывается `onBack()`

### 5. Автоматическое скрытие всплывающего меню

#### Проблема
- Всплывающее меню оставалось открытым и могло перекрывать контент
- Не было способа автоматически его закрыть

#### Решение
**Файл: `app/src/main/java/com/example/ohrana/JournalScreen.kt`**

**Изменения:**
- Добавлен `LaunchedEffect` с `delay(3000)` для автоматического скрытия меню через 3 секунды
- Добавлен импорт `kotlinx.coroutines.delay`

```kotlin
LaunchedEffect(showMenu) {
    if (showMenu) {
        delay(3000)
        showMenu = false
    }
}
```

#### Результат
- Всплывающее меню автоматически скрывается через 3 секунды после появления
- Меню также скрывается при нажатии системной кнопки "Назад"
