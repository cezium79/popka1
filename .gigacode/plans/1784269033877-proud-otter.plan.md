# План исправления ошибок в JournalScreen.kt

## Суть задачи
Исправить ошибку FATAL EXCEPTION: main, вызванную вложением прокручиваемых контейнеров и ошибкой парсинга дат.

## Проблемы

### 1. Вложенные прокручиваемые контейнеры
- `LazyVerticalGrid` (календарь) был вложен внутри `Column` с `Modifier.verticalScroll()`
- Это создавало бесконечные ограничения по высоте, что запрещено в Compose
- Ошибка: `Vertically scrollable component was measured with an infinity maximum height constraints`

### 2. ParseException при парсинге дат
- `storageFormat` использовал неверный формат даты
- База данных хранит даты в формате "dd.MM.yyyy HH:mm:ss", а не "yyyy-MM-dd HH:mm:ss"

## Решения

### 1. Исправление вложенных прокручиваемых контейнеров (ФИНАЛЬНОЕ)
- Убрал `Column` с `Modifier.verticalScroll()`
- Переписал макет, используя только `LazyColumn` для всего контента
- Удалил использование `LazyVerticalGrid` (вложенный Lazy* контейнер)
- Вместо `LazyVerticalGrid` используется `LazyColumn` с обычными `Row` для отображения недель
- Каждая неделя - это отдельная строка из 7 дней с `Modifier.weight(1f)`

### 2. Исправление формата даты
- Изменил `storageFormat` с "yyyy-MM-dd HH:mm:ss" на "dd.MM.yyyy HH:mm:ss"
- Добавил защиту try-catch при парсинге дат
- Добавил проверку на пустые значения `startTime` и `endTime`

### 3. Дополнительные изменения
- Переписал функцию `getDaysInMonth()` в `getWeeksInMonth()` для разбиения дней на недели
- Теперь календарь отображается построчно, без использования `LazyVerticalGrid`

## Файлы для модификации

### 1. Модифицировать: `app/src/main/java/com/example/ohrana/JournalScreen.kt`
- Строка 62: исправить `storageFormat` на правильный формат
- Переписать структуру макета для использования только `LazyColumn`
- Удалить `LazyVerticalGrid` и заменить на Row-based календарь
- Добавить защиту при парсинге дат
