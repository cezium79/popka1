# Реализация групповой работы охранников

## Обновленная 06.07.2026

### Функциональность

#### 1. Выбор количества охранников в админ-панели
В разделе "Панель администратора" добавлено выпадающее меню для выбора количества охранников:
- **"1 охранник (старший)"** - один охранник (стандартный режим)
- **"Старший и 1 охранник"** - групповая работа из 2 человек
- **"Старший и 2 охранника"** - групповая работа из 3 человек

#### 2. Экран "privet" (приветствие)

**Для guardsCount = 1:**
- Показывается текст "Поднесите личную карту к телефону"
- При NFC-сканировании сразу начинается смена
- Кнопка "ОТКРЫТЬ СМЕНУ" отсутствует

**Для guardsCount > 1 и закрытой смены:**
- Скрыт текст про NFC
- Показана кнопка **"ОТКРЫТЬ СМЕНУ"** внизу экрана
- При нажатии открывается экран "GuardNfcSelectionScreen"

**Для guardsCount > 1 и открытой смены:**
- Кнопка "ОТКРЫТЬ СМЕНУ" скрыта
- При NFC-сканировании сразу открывается экран "Обходы за смену"

#### 3. Экран GuardNfcSelectionScreen (выбор охранников через NFC)

Новый экран для групповой работы:
- Показывает количество охранников, которое нужно выбрать
- Добавляет кнопку "Добавить охранника (NFC)"
- При нажатии на кнопку активируется NFC-сканирование
- При поднесении NFC-метки автоматически добавляется охранник из списка сотрудников
- Автоматически отключает сканирование после добавления последнего охранника
- Показывает список выбранных охранников с возможностью удаления
- Проверяет наличие "старшего смены" в списке
- При достижении нужного количества появляется кнопка "НАЧАТЬ СМЕНУ"

#### 4. Экран RoundsScreen (обходы за смену)

Обновлен для отображения списка охранников:
- Для **1 охранника**: показывает имя сотрудника
- Для **группы охранников**: показывает карточку со списком:
  - ФИО каждого охранника
  - Роль каждого охранника (старший смены/охранник)

### Архитектура

#### Новые файлы
- **GuardNfcSelectionScreen.kt** - экран выбора охранников через NFC-сканирование

#### Обновленные файлы
- **MainActivity.kt** - обновлена логика экрана "privet", добавлен экран "guard_nfc_selection"
- **AdministratorScreen.kt** - добавлено выпадающее меню для выбора количества охранников
- **RoundsScreen.kt** - обновлено отображение списка охранников

### Режимы работы

#### Режим 1: Стандартный (guardsCount = 1)
- Простой NFC-скан
- Смена начинается сразу после сканирования
- Работает один охранник

#### Режим 2: Групповой (guardsCount > 1)
- На экране приветствия кнопка "ОТКРЫТЬ СМЕНУ"
- Открывается GuardNfcSelectionScreen
- Последовательное добавление охранников через NFC
- Обязательное наличие "старшего смены" в списке
- После выбора всех охранников - старт смены
- В экране "Обходы за смену" отображается список всех охранников

### Технические детали

#### GuardMember Model
```kotlin
data class GuardMember(
    val nfcId: String,          // NFC ID охранника
    val name: String,           // Имя охранника
    val role: String,           // Роль: "старший смены" или "охранник"
    val startTime: String? = null  // Время начала работы
)
```

#### SharedPrefsManager Methods
- `getGuardsCount()` - получить количество охранников
- `setGuardsCount(count: Int)` - установить количество охранников
- `loadGuards()` - загрузить список охранников
- `saveGuards(guards: List<GuardMember>)` - сохранить список охранников
- `getShiftLeader()` - получить старшего смены
- `getCurrentGuard()` - получить текущего охранника
- `setCurrentGuard(nfcId: String)` - установить текущего охранника

### NFC-сканирование

При работе с группой охранников:
1. Активируется reader mode при открытии GuardNfcSelectionScreen
2. При поднесении NFC-метки автоматически ищется сотрудник в списке
3. Добавляется GuardMember с данными сотрудника
4. Сканирование останавливается автоматически после добавления нужного количества
5. NFC-сканирование работает только для добавления охранников (не для чекпоинтов)

### Обратная совместимость

- При guardsCount = 1 поведение остается прежним (стандартный NFC-скан)
- Существующие функции (guard_selection через список) работают как и раньше
- База данных ShiftRecord обновлена для хранения guardList
- Все отчеты и логи корректно работают с группами
