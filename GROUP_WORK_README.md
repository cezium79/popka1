# Групповая работа охранников

## Описание функции

В приложении добавлена полная поддержка работы нескольких охранников на одной смене. Теперь администратор может настроить количество охранников (1, 2 или 3), а система автоматически управляет выбором охранников через NFC-сканирование и переключением между ними при прохождении обходов.

## Новые функции

### 1. Выбор количества охранников

**Где:** Экран администратора → "Количество охранников"

**Описание:** Администратор может выбрать количество охранников из выпадающего меню.

**Доступные варианты:**
- "1 охранник (старший)" - стандартный режим работы одного охранника
- "Старший и 1 охранник" - группа из 2 охранников
- "Старший и 2 охранника" - группа из 3 охранников

**Интерфейс:** Выпадающее меню (DropdownMenu) с сохранением выбора.

### 2. Экран выбора охранников (GuardNfcSelectionScreen)

**Где:** При нажатии на кнопку "ОТКРЫТЬ СМЕНУ" на экране приветствия при настроенном количестве охранников > 1

**Описание:** Автоматическое NFC-сканирование для выбора всех охранников, которые будут работать в этой смене.

**Функционал:**
- NFC-сканирование запускается автоматически при открытии экрана
- Показывает текущий этап сканирования:
  - "Поднесите карту Старшего смены" - первый этап
  - "Поднесите карту охранника" - второй этап
- Сообщения об успешном сканировании:
  - "Старший смены: Иванов Иван"
  - "Охранник: Петров Петр"
- Сообщения об ошибках:
  - "Охранник уже выбран"
  - "Старший смены уже выбран"
  - "В штате нет такого сотрудника"
  - "Достигнут лимит охранников"
- Отображение уже выбранных охранников с возможностью удаления
- Кнопка "НАЧАТЬ СМЕНУ" активируется только после выбора всех охранников

### 3. Структура данных для группы охранников

**Новая модель:** `GuardMember`
```kotlin
data class GuardMember(
    val nfcId: String,      // NFC ID охранника
    val name: String,       // Имя охранника
    val role: String,       // Роль: "старший смены" или "охранник"
    val startTime: String? = null  // Время начала работы
)
```

**Хранение:** Список охранников сохраняется в SharedPreferences с ключом `guards_list`.

### 4. Обновленная модель смены

**Изменения в `ShiftRecord`:**
- Добавлено поле `guardList: List<GuardMember>` для хранения списка всех охранников на смене
- Поле `employeeName` сохранено для совместимости (содержит имя первого охранника)

### 3. Методы SharedPrefsManager

**Новые методы:**
- `getGuardsCount(): Int` - получить количество охранников
- `setGuardsCount(count: Int)` - установить количество охранников
- `loadGuards(): List<GuardMember>` - загрузить список охранников
- `saveGuards(guards: List<GuardMember>)` - сохранить список охранников
- `getCurrentGuard(): GuardMember?` - получить текущего охранника
- `setCurrentGuard(nfcId: String)` - установить текущего охранника
- `getShiftLeader(): GuardMember?` - получить старшего смены
- `getRoundCheckpointIndex(roundIndex: Int): Int` - получить индекс чекпоинта для конкретного обхода
- `isRoundStarted(roundIndex: Int): Boolean` - проверить, запущен ли обход (checkpointIndex > 0)
- `getRoundStartTime(roundIndex: Int): String` - получить время начала конкретного обхода
- `getRoundEndTime(roundIndex: Int): String` - получить время завершения конкретного обхода
- `isRoundCompleted(roundIndex: Int): Boolean` - проверить, завершен ли обход

**Логика индексов:**
- Индекс чекпоинта сохраняется отдельно для каждого обхода: `round_${roundIndex}_checkpoint_index`
- При запуске обхода индекс НЕ сбрасывается (сохраняется при возврате)
- При завершении обхода индекс сохраняется
- При закрытии всей смены все индексы сбрасываются

### 6. Обновленная база данных смен

**Изменения в `ShiftDatabaseManager`:**
- Метод `startNewShift()` теперь принимает список `guardList: List<GuardMember>`
- Метод `loadAllShifts()` загружает список охранников из сохраненных данных
- Метод `saveShift()` сохраняет список охранников

## Порядок работы

### Для администратора:

1. Войдите в панель администратора
2. Перейдите в настройки
3. Укажите количество охранников (1-5)
4. Сохраните настройки

### Для охранников:

1. Охранник прикладывает NFC-карту на экран "Привет"
2. Если количество охранников > 1, открывается экран выбора охранников
3. Выбираются все охранники, которые будут работать в этой смене
4. При групповой работе обязательно должен быть выбран старший смены (указать роль "старший смены" при добавлении сотрудника)
5. Нажимается кнопка "НАЧАТЬ СМЕНУ"
6. Происходит старт смены со всеми выбранными охранниками
7. Все охранники могут проходить обходы по очереди

## Смена охранника во время работы

Текущий охранник, который проходит обход, определяется по NFC ID, который устанавливается при начале смены. Для смены текущего охранника в кабинете охранника необходимо добавить функционал переключения между охранниками (может быть реализован в будущем).

## Совместимость

Все изменения обратно совместимы:
- Если количество охранников = 1, работает старая логика
- Старые смены без списка охранников загружаются корректно
- Метод `startNewShift(employeeName: String, strictSequenceEnabled: Boolean)` сохранен для обратной совместимости

## Технические детали

### Файлы, которые были изменены:

1. `ShiftDatabaseModels.kt` - добавлена модель `GuardMember`, обновлена `ShiftRecord`
2. `SharedPrefsManager.kt` - добавлены методы для работы с группой охранников
3. `ShiftDatabaseManager.kt` - обновлены методы для сохранения/загрузки списка охранников
4. `AdministratorScreen.kt` - добавлен UI для выбора количества охранников
5. `MainActivity.kt` - добавлен экран выбора охранников, обновлены логики переходов
6. `GuardSelectionScreen.kt` - новый экран для выбора охранников

### NFC-теги

Каждый охранник должен иметь уникальный NFC ID в списке сотрудников. При сканировании NFC-тега система проверяет, существует ли сотрудник и добавляет его в список выбранных охранников.
