# Изменения в системе фиксации происшествий

## Описание проблемы
При создании фото происшествия комментарий к фото не сохранялся корректно. Если комментарий не создавался, в HTML отчете отображалось пустое описание вместо "No Comments".

## Решение

### 1. Создан IncidentArgumentManager (IncidentArgumentManager.kt)

Новый менеджер для хранения аргументов происшествия во временных JSON файлах. Хранит:
- ID смены
- ID обхода
- Имя сотрудника
- Время и дата
- Тип происшествия
- Описание
- Путь к фото
- Координаты (GPS)
- Информация о чекпоинте

Функции:
- `createIncidentArgs()` - создает новую запись аргументов
- `loadIncidentArgs()` - загружает аргументы по ID
- `updateDescription()` - обновляет описание
- `updatePhotoPath()` - обновляет путь к фото
- `updateCoordinates()` - обновляет GPS координаты
- `updateCheckpointInfo()` - обновляет информацию о чекпоинте
- `deleteIncidentArgs()` - удаляет аргументы
- `loadActiveIncidentArgs()` - загружает активные аргументы для смены
- `clearAllIncidentArgs()` - очищает все аргументы

### 2. Обновлен IncidentCaptureScreen.kt

Изменена логика работы с фото и комментариями:

#### При нажатии "Комментарий":
1. Создается временный файл фото
2. Создаются аргументы происшествия с пустым описанием
3. Открывается диалог для редактирования типа и описания
4. При нажатии "OK":
   - Обновляется описание в аргументах
   - Копируется фото в галерею
   - Создается запись в базе данных с полным описанием
   - Удаляются временные аргументы

#### При нажатии "Сохранить":
1. Копируется фото в галерею
2. Создается запись в базе данных с пустым описанием
3. В HTML отчете пустое описание будет отображаться как "No Comments"
4. Удаляются временные аргументы

#### При нажатии "Повторить":
1. Удаляется временный файл фото
2. Удаляются аргументы происшествия
3. Можно начать заново

#### При выходе из экрана (BackHandler):
1. Удаляются все несохраненные аргументы
2. Никаких временных файлов не остается

### 3. Обновлен HTML отчет

В разделе "Происшествия" теперь:
- Если описание пустое → отображается "No Comments"
- Если описание есть → отображается текст комментария

```kotlin
<span><label>Описание:</label> ${if (incident.description.isNullOrBlank()) "No Comments" else incident.description}</span>
```

## Поведение

### Сценарий 1: Фото без комментария (кнопка "Сохранить")
- Фото сохраняется в галерею
- Создается запись с пустым описанием
- В HTML отчете: "Описание: No Comments"

### Сценарий 2: Фото с комментарием (кнопка "Комментарий")
- Фото сохраняется во временный файл
- Открывается диалог для редактирования
- При нажатии "OK":
  - Обновляется описание
  - Фото копируется в галерею
  - Создается запись с полным описанием
- В HTML отчете отображается введенный текст

### Сценарий 3: Повторное создание фото (кнопка "Повторить")
- Временный файл удаляется
- Аргументы удаляются
- Можно сделать новое фото

## Файлы

### Новые файлы:
- `IncidentArgumentManager.kt` - менеджер аргументов происшествия

### Измененные файлы:
- `IncidentCaptureScreen.kt` - обновлена логика работы с фото и комментариями
- `CloudStorageManager.kt` - обновлен HTML отчет для отображения "No Comments"
- `ShiftDatabaseManager.kt` - добавлена функция обновления последнего происшествия (возможно не используется в новой версии)

## Тестирование

Для проверки работы:

1. **Без комментария**:
   - Сделать фото
   - Нажать "Сохранить"
   - Открыть HTML отчет
   - Проверить, что описание = "No Comments"

2. **С комментарием**:
   - Сделать фото
   - Нажать "Комментарий"
   - Ввести текст
   - Нажать "OK"
   - Открыть HTML отчет
   - Проверить, что описание = введенному тексту

3. **Повторное создание**:
   - Сделать фото
   - Нажать "Повторить"
   - Убедиться, что старое фото удалено
