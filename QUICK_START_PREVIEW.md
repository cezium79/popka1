# Как работать с дизайном окон в Android Studio

## 1. Открыть Preview-окно

### Вариант А: Кнопка Preview
1. Откройте файл Composable-функции (`.kt`)
2. Найдите кнопку **"Preview"** справа в верхней части редактора
3. Кликните на неё - откроется вкладка с предпросмотром

### Вариант Б: Навигация
1. Откройте файл `Marshruti.kt`
2. Найдите функцию `MarshrutiScreenPreview()` в конце файла
3. Кликните на кнопку `Preview` рядом с аннотацией `@Preview`
4. Откроется предпросмотр экрана настройки маршрутов

## 2. Что можно делать в Preview

### Изменить устройство
Сверху в Preview-окне есть выпадающий список:
- Pixel 5, Pixel 6, Pixel 7
- Tablet ( Nexus 7, Nexus 10)
- Foldable (Pixel Fold)
- Custom (свой размер экрана)

### Изменить тему
В меню Preview найдите переключатель темы:
- **Light** - светлая тема (Material Design по умолчанию)
- **Dark** - темная тема
- **System** - использует системную тему устройства

### Поворот экрана
Нажмите кнопку **Rotate** (🔄) для смены ориентации:
- Portrait (вертикальная)
- Landscape (горизонтальная)

### Масштабирование
- Используйте ползунок масштаба внизу Preview-окна
- От 50% до 200%

### Изменить язык
Нажмите кнопку **Locales** (🌐)
- Выберите язык интерфейса
- Проверьте, как отображаются русские тексты

## 3. Типичные настройки Preview

```kotlin
@Preview(
    showBackground = true,              // Показывать фон
    name = "Название экрана",           // Имя в списке preview
    backgroundColor = 0xff000000,       // Цвет фона (hex)
    uiMode = Configuration.UI_MODE_NIGHT_YES,  // Темная тема
    locale = "ru"                       // Русский язык
)
@Composable
fun MyScreenPreview() {
    MyScreen()
}
```

## 4. Когда Preview НЕ РАБОТАЕТ

Preview **НЕ ОТОБРАЖАЕТ** экраны, которые:

❌ Используют `LocalContext.current`  
❌ Требуют реальный `Context` для инициализации  
❌ Используют `SharedPrefsManager`, `AlarmScheduler` и другие менеджеры  
❌ Зависят от CameraX или других системных сервисов  
❌ Используют `remember { ... }` с реальными объектами

**Ваш проект имеет такие экраны:**
- `PhotoCaptureScreen` - не работает Preview (использует `LocalContext`)
- `OhrannikCabinetScreen` - не работает Preview (использует реальный контекст)
- `MarshrutiScreen` - **работает Preview** (использует `remember`)

## 5. Рекомендации по дизайну

### Размеры отступов
```
4.dp  - маленький отступ
8.dp  - стандартный отступ
16.dp - большой отступ
24.dp - очень большой отступ
```

### Размеры текста
```
24.sp - заголовок раздела
18.sp - подзаголовок
16.sp - основной текст
14.sp - второстепенный текст
12.sp - мелкий текст
```

### Размеры кнопок
```
48.dp  - минимальный размер кнопки (48x48 dp)
56.dp  - высота кнопки Material Design
60.dp  - большая кнопка действия
```

### Углы
```
RoundedCornerShape(8.dp)  - маленькие скругления
RoundedCornerShape(12.dp) - средние скругления
RoundedCornerShape(28.dp) - большие скругления (кнопки)
```

## 6. Быстрое изменение дизайна

1. **Измените код** в Composable-функции
2. **Сохраните файл** (Ctrl+S)
3. **Проверьте Preview** - изменения отобразятся автоматически
4. **Если Preview не обновился** - нажмите **Refresh** (🔄) в Preview-окне

## 7. Пример изменения дизайна

### Было:
```kotlin
Column(
    modifier = Modifier
        .fillMaxWidth()
        .padding(16.dp)
) {
    Text("Заголовок")
    Spacer(modifier = Modifier.height(16.dp))
    Text("Текст")
}
```

### Стало:
```kotlin
Column(
    modifier = Modifier
        .fillMaxWidth()
        .padding(24.dp)  // Увеличили отступы
        .background(Color(0xFFE3F2FD))  // Добавили фон
) {
    Text("Заголовок", fontSize = 24.sp)  // Увеличили шрифт
    Spacer(modifier = Modifier.height(24.dp))
    Text("Текст", fontSize = 18.sp)
}
```

Preview автоматически обновится и вы увидите изменения!

## 7. Новая функциональность: Замена картинки прибора

### В пункте 4 "Картинки приборов для фото" теперь доступна замена:

1. **Нажмите на иконку "+"** рядом с привязанной картинкой
2. **Откроется диалог** с двумя кнопками:
   - **Заменить** - выберите новую картинку из галереи
   - **Удалить** - удалить текущую картинку

### Как это работает:

```
Текущие привязки:
CP-001 [+] [🗑️]
CP-002 [+] [🗑️]
```

- Кнопка **[+]** (плюс) - открывает диалог замены/удаления
- Кнопка **[🗑️]** (корзина) - удаляет картинку сразу

При клике на **[+]** показывается диалог:
```
Управление картинкой
Чекпоинт: CP-001

[Картинка прибора здесь]

Картинка привязана. Нажмите 'Заменить', чтобы выбрать новую,
или 'Удалить', чтобы убрать привязку.

[Заменить] [Удалить]
```

Теперь вы сразу видите текущую картинку и можете принять решение, нужно ли её менять!

**Исправление:** Теперь используется `LaunchedEffect` для загрузки картинки и отдельное состояние `dialogBitmap` для хранения загруженного изображения, что предотвращает вылеты приложения.

## 8. Отладка дизайна

### Точки останова
- Добавьте `debugInspectorInfo { }` для отладки модификаторов
- Используйте `Modifier.inspectable()` для детального анализа

### Логирование
- Используйте `Modifier.then(Modifier.debugInspectorInfo { ... })`
- Проверьте в Preview, что модификаторы применяются правильно

### Цвета
- Используйте `Color(0xFFRRGGBB)` для точных цветов
- Или `Color(red, green, blue)` для RGB-значений
- Или `Color.HSL(hue, saturation, lightness)` для HSL

## 9. Проверка на разных устройствах

1. Выберите разные устройства в Preview (Phone, Tablet, Foldable)
2. Проверьте, как интерфейс адаптируется
3. Убедитесь, что текст не обрезается
4. Проверьте, что кнопки доступны для касания

## 10. Финальная проверка

После изменений дизайна:
1. ✅ Соберите проект: `./gradlew assembleDebug`
2. ✅ Установите на устройство: `./gradlew installDebug`
3. ✅ Протестируйте на реальном устройстве
4. ✅ Проверьте на разных размерах экрана

## 8. Частые проблемы и решения

### Проблема: Preview не отображается
**Решение:** Проверьте, что функция помечена `@Composable`

### Проблема: Preview не обновляется
**Решение:** Нажмите **Refresh** или перезапустите Android Studio

### Проблема: Preview показывает ошибку
**Решение:** Проверьте консоль ошибок справа - там будет описание проблемы

### Проблема: Экран выглядит иначе на устройстве
**Решение:** Preview показывает упрощённую версию - всегда тестируйте на реальном устройстве
